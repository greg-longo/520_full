Course Overview

DTSC 520 is an introductory graduate-level course in data science. The course is designed for students who want to build practical, working skills in Python-based data science - not just theoretical knowledge, but the ability to pick up a real dataset and do something useful with it.

By the end of the course you will be able to write Python programs to manipulate and analyze data, create visualizations that communicate findings clearly, use version control to manage your code professionally, and think critically about data problems - asking the right questions before touching a dataset.

Learning outcomes

  • Understand the data science landscape: roles, tools, and how DS fits within organizations
  • Write Python programs using variables, control flow, data structures, and functions
  • Use NumPy for efficient numerical computation and array manipulation
  • Use pandas to load, clean, filter, and aggregate tabular datasets
  • Create charts with Matplotlib and Seaborn that accurately represent your data
  • Use Git and GitHub for version control and collaborative code management
  • Apply critical thinking to evaluate data quality, question framing, and analytical conclusions

How this course works

Each module has three components: instructional content (the pages you are reading now), hands-on simulations that put you in realistic data science scenarios, and a 10-question quiz to test your understanding. Work through the content, try the simulation, then take the quiz. The simulations are the most important part - reading about pandas is not the same as using it to clean a messy dataset.

A note on difficulty
This is an introductory course, but "introductory" does not mean easy. Graduate-level means you are expected to struggle productively with unfamiliar material, look things up, and work through confusion rather than waiting for it to be resolved for you. That is what real data scientists do every day.

Data Science in Action

Data science is not an abstract discipline. It is being applied right now to every industry you can name. These examples are drawn from real applications - the same techniques you will learn in this course power each of them.

Sports Analytics

Player Evaluation & Scouting

MLB teams use pandas and statistical modeling to evaluate prospects from minor league data. The Moneyball revolution (Oakland A's, 2002) showed that data could find market inefficiencies in player valuation. Today every major team employs data scientists.

Healthcare

Early Disease Detection

Machine learning models trained on medical imaging data can detect diabetic retinopathy, lung cancer, and skin lesions with accuracy matching or exceeding specialist physicians. NumPy and Python underpin the entire image processing pipeline.

Finance

Fraud Detection

Credit card companies process billions of transactions daily, flagging potentially fraudulent ones in real time using anomaly detection models. The pandas cleaning and feature engineering you will learn in Module 4 is the foundation of this pipeline.

Retail / E-Commerce

Recommendation Engines

"Customers who bought X also bought Y." These systems are trained on purchase history matrices using collaborative filtering - a technique built on NumPy array operations. Amazon estimates 35% of its revenue comes from recommendations.

Climate Science

Climate Modeling

Climate scientists use Python (NumPy, pandas, Matplotlib) to process satellite data, model atmospheric conditions, and communicate findings. The visualizations you see in IPCC reports are generated with the tools from Module 5.

Public Policy

Predictive Policing & Bias Detection

Government agencies use data science for resource allocation, but the field also grapples with ethical failures - biased training data producing discriminatory models. Critical thinking (Module 1) is as important as technical skill.

The common thread
Every example above uses the same core stack: Python for scripting, pandas/NumPy for data manipulation, Matplotlib/Seaborn for visualization, and Git for version control. Mastering these seven modules gives you the foundation to enter any of these fields.

Software Setup

You need two things to run Python data science code: a Python environment and a way to write and run code interactively. The recommended setup for this course is Anaconda + Jupyter Lab.

Option 1: Anaconda (recommended)

Anaconda is a Python distribution that bundles Python, Jupyter, and all the major data science libraries (NumPy, pandas, Matplotlib, Seaborn) in a single installer. It is the easiest way to get started.

  1. Download Anaconda. Go to anaconda.com/download and download the installer for your operating system (Windows, macOS, or Linux). Choose the Python 3 version.
  2. Run the installer. Accept defaults. On Windows, check "Add Anaconda to PATH" if prompted (makes command-line use easier). The install takes 5-10 minutes.
  3. Open Jupyter Lab. After installation, open the Anaconda Navigator app and click "Launch" under JupyterLab. Or, from the terminal/command prompt, run: jupyter lab
  4. Verify your install. In a new Jupyter notebook, run the cell below. If it prints version numbers without errors, you are ready.
# Run this in a Jupyter cell to verify your setup
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns

print("NumPy:",    np.__version__)
print("pandas:",   pd.__version__)
print("Matplotlib:", plt.matplotlib.__version__)
print("Seaborn:",  sns.__version__)

Option 2: Google Colab (no install required)

If you cannot install software on your machine, Google Colab (colab.research.google.com) runs Jupyter notebooks in your browser with all major libraries pre-installed. You just need a Google account. Files are saved to Google Drive.

VS Code users
If you prefer VS Code as your editor, install the Python extension and the Jupyter extension. You can then run .ipynb notebook files directly inside VS Code. This is a good setup once you are comfortable, but Jupyter Lab is simpler to start with.

Verifying Git is installed

# Run in your terminal (not Jupyter)
git --version
# Should print: git version 2.x.x

Course Resources

These are the primary resources for the course. The textbooks are not required purchases - both are available free online in their full versions.

Textbooks

  • "Learn Python, Break Python" - The primary reference for Modules 2-4. Covers Python fundamentals through pandas in a practical, project-based style. Chapters align directly with module content.
  • "Build a Career in Data Science" - The reference for Module 1. Covers the DS landscape, roles, job searching, and how to position yourself for a data career. Available free online.

Key documentation sites

  • pandas docs - pandas.pydata.org/docs - The official reference. The "10 minutes to pandas" guide is excellent.
  • NumPy docs - numpy.org/doc - Especially the "NumPy for absolute beginners" guide.
  • Matplotlib gallery - matplotlib.org/stable/gallery - Browse chart examples and copy the code. The fastest way to figure out how to make a specific chart.
  • Seaborn docs - seaborn.pydata.org - Each function page has a visual example gallery.

When you get stuck

Every working data scientist uses Stack Overflow and the official documentation daily. Getting good at reading documentation and searching for solutions is itself a core skill. When you hit an error: read the error message carefully (it usually tells you exactly what went wrong), search the exact error message in quotes, check the official docs for the function you are using.

How to read a Python error
Python error messages are called tracebacks. Read them from the bottom up. The last line is the actual error (e.g., KeyError: 'avg'). The lines above show where in your code the error originated. The bottom line is the one that matters most.
