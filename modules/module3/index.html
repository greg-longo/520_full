NumPy: Arrays and Vectorized Operations

The ndarray: NumPy's Core Object

NumPy's central data structure is the ndarray (n-dimensional array) - a fixed-type, fixed-size grid of values. Unlike a Python list, every element in a NumPy array is the same data type, and they are stored contiguously in memory. This is what makes NumPy so fast.

import numpy as np

# Create a 1D array from a list
batting_avgs = np.array([.312, .287, .301, .268, .295])

# Key attributes
batting_avgs.shape    # (5,) - one dimension, 5 elements
batting_avgs.dtype    # float64
batting_avgs.ndim     # 1
batting_avgs.size     # 5 - total number of elements

# 2D array: rows = players, cols = [avg, hr, rbi]
stats = np.array([
    [.312, 4, 22],
    [.287, 2, 15],
    [.301, 7, 31],
    [.268, 1, 9]
])
stats.shape    # (4, 3) - 4 rows, 3 columns
output — batting_avgs / stats arrays
batting_avgs: [0.312 0.287 0.301 0.268 0.295] dtype: float64 stats (4x3): [[0.312 4. 22.] [0.287 2. 15.] [0.301 7. 31.] [0.268 1. 9.]] stats.shape: (4, 3) stats.dtype: float64

Creating arrays

# Arrays filled with zeros or ones
np.zeros(5)           # [0. 0. 0. 0. 0.]
np.ones((3, 4))       # 3x4 matrix of 1.0

# Range of values (like Python range, but as an array)
np.arange(0, 10, 2)  # [0 2 4 6 8]

# Evenly spaced values between two endpoints
np.linspace(0, 1, 5) # [0.   0.25  0.5  0.75  1.  ]

# Random arrays (useful for simulation and testing)
np.random.rand(4)        # 4 uniform random floats in [0, 1)
np.random.randn(4)       # 4 standard normal random values
np.random.randint(1, 10, size=5)  # 5 random ints in [1, 10)

Reshaping arrays

# Reshape without changing data
flat = np.arange(12)        # [0  1  2 ... 11]
grid = flat.reshape(3, 4)   # 3x4 matrix
col  = flat.reshape(-1, 1) # -1 = "infer this dimension"

# Flatten back to 1D
grid.flatten()
grid.ravel()  # faster - returns a view when possible
Why not just use Python lists?
Python lists can hold mixed types, but that flexibility costs speed. NumPy stores values of one type in a single contiguous memory block, allowing C-level loops instead of Python-level loops. For large datasets, NumPy operations can be 10-100x faster than the equivalent Python list comprehension.

Indexing and Slicing

NumPy indexing is similar to Python list slicing but extends to multiple dimensions. Slicing returns a view of the original data, not a copy - so modifying the slice modifies the original array too.

avgs = np.array([.312, .287, .301, .268, .295])

# Basic indexing (0-based)
avgs[0]      # 0.312  - first element
avgs[-1]     # 0.295  - last element

# Slicing: start:stop:step (stop is exclusive)
avgs[1:4]    # [.287, .301, .268]  - indices 1, 2, 3
avgs[:3]     # [.312, .287, .301]  - first 3
avgs[::-1]   # reversed array

2D indexing

# stats array: rows = players, cols = [avg, hr, rbi]
stats = np.array([
    [.312, 4, 22],
    [.287, 2, 15],
    [.301, 7, 31],
    [.268, 1, 9]
])

# Single element: row, col
stats[0, 1]     # 4.0  (Rivera's HRs)

# Full row
stats[2]        # [.301, 7, 31]  (Walker)

# Full column: all rows, column index 0
stats[:, 0]    # [.312, .287, .301, .268]  (all batting avgs)


# Submatrix: first 2 players, first 2 stats
stats[:2, :2]  # [[.312, 4], [.287, 2]]
output — 2D indexing examples
stats[0, 1] → 4.0 # Rivera's HRs stats[2] → [0.301 7. 31.] # Walker's full row stats[:, 0] → [0.312 0.287 0.301 0.268] # all batting avgs stats[:2, :2] → [[0.312 4.] [0.287 2.]]
Views vs. copies
NumPy slices return views, not copies. If you do sub = stats[:2] and then sub[0, 0] = 0, you will also change stats[0, 0]. To get an independent copy, call .copy(): sub = stats[:2].copy().

Vectorization: No Loops Required

The most important NumPy concept is vectorization - applying an operation to every element of an array at once, without writing a Python loop. This both simplifies your code and makes it dramatically faster because the loop runs in compiled C code instead of the Python interpreter.

Python loop (slow)

Iterates in the Python interpreter. Each element requires a separate Python object operation.

For 1 million elements, this creates 1 million Python overhead events.

NumPy vectorized (fast)

Passes the entire array to a C routine. The loop runs at C speed with no Python object overhead.

For 1 million elements, this is typically 10-100x faster.

avgs = np.array([.312, .287, .301, .268])
salaries = np.array([16800, 14400, 19200, 14400])

# Arithmetic - applied to every element
avgs * 1000          # [312.  287.  301.  268.]
avgs + 0.050         # add .050 to all
salaries / 52        # weekly pay for all players

# Element-wise operations between two arrays
avgs * salaries      # each player's avg x salary

# Broadcasting: scalar vs. array (as above)
# Broadcasting: 1D vs. 2D (NumPy auto-expands dims)
stats = np.array([[.312, 4], [.287, 2]])
weights = np.array([100, 1])
stats * weights      # [[31.2, 4], [28.7, 2]]

# Universal functions (ufuncs) - vectorized math
np.sqrt(avgs)        # square root of each element
np.log(salaries)     # natural log (useful for skewed data)
np.abs(avgs - .300)  # distance from .300 for each player
output — vectorized operations
avgs * 1000 → [312. 287. 301. 268.] avgs + 0.050 → [0.362 0.337 0.351 0.318] salaries / 52 → [323.08 276.92 369.23 276.92] # weekly pay np.abs(avgs - .300) → [0.012 0.013 0.001 0.032] stats * weights → [[31.2 4.] [28.7 2.]]

Aggregating along axes

stats = np.array([
    [.312, 4, 22],
    [.287, 2, 15],
    [.301, 7, 31]
])

# No axis: collapse everything to one number
stats.sum()           # total of all 9 values

# axis=0: collapse rows - result has shape (3,) = one value per column
stats.sum(axis=0)    # [.900, 13, 68] - column totals
stats.mean(axis=0)  # [.300, 4.33, 22.67] - column means

# axis=1: collapse columns - result has shape (3,) = one value per row
stats.sum(axis=1)   # [26.312, 17.287, 38.301] - row totals
output — axis aggregations on stats (3x3 array)
stats.sum() → 102.9 # grand total stats.sum(axis=0) → [ 0.900 13.000 68.000] # column totals stats.mean(axis=0) → [ 0.300 4.333 22.667] # column means stats.sum(axis=1) → [26.312 17.287 38.301] # row totals (per player)

Statistical Methods

NumPy provides a comprehensive set of statistical functions. These are the building blocks for exploratory data analysis before you even reach pandas.

FunctionDescriptionExample
np.mean(a)Arithmetic meanMean batting avg
np.median(a)Middle value (50th percentile)Median salary
np.std(a)Standard deviationSpread of batting avgs
np.var(a)Variance (std squared)Variance in HR counts
np.min(a) / np.max(a)Minimum / maximumLowest/highest avg
np.sum(a)Sum of all elementsTotal RBI count
np.percentile(a, q)qth percentile value75th percentile salary
np.argmin(a) / np.argmax(a)Index of min/max valueWho hit the most HRs?
np.cumsum(a)Cumulative sumRunning RBI total
np.unique(a)Sorted unique valuesUnique positions in roster
 
salaries = np.array([16800, 14400, 19200, 760000])

print(f"Mean:   {np.mean(salaries):.0f}")    # 202600
print(f"Median: {np.median(salaries):.0f}")  # 18000
print(f"Std:    {np.std(salaries):.0f}")    # ~370000

# Who has the highest salary?
idx = np.argmax(salaries)   # 3
players = np.array(['Rivera', 'Chen', 'Walker', 'Kim'])
print(players[idx])          # 'Kim'

# Percentile analysis
np.percentile(salaries, 25)   # 25th percentile
np.percentile(salaries, [25, 50, 75])  # quartiles

Boolean Indexing

Boolean indexing lets you select elements based on a condition. A comparison on a NumPy array produces a boolean array of the same shape. Using that boolean array as an index returns only the elements where the condition is True.

avgs = np.array([.312, .287, .301, .268, .295])

# Step 1: comparison creates a boolean array
mask = avgs > .295
# array([ True, False,  True, False, False])

# Step 2: use the mask as an index
avgs[mask]     # [.312, .301]

# Same thing in one line
avgs[avgs > .295]

# Combine conditions with & (and), | (or), ~ (not)
avgs[(avgs > .285) & (avgs < .310)]  # [.287, .301, .295]

# Use boolean mask to select from a DIFFERENT array
players = np.array(['Rivera', 'Chen', 'Walker', 'Kim', 'Jones'])
players[avgs > .295]   # ['Rivera', 'Walker']

# Modify values matching a condition
avgs[avgs < .280] = .280   # floor all avgs at .280
output — boolean indexing
mask = avgs > .295 → [ True False True False False] avgs[mask] → [0.312 0.301] avgs[(avgs>.285)&(avgs<.310)] → [0.287 0.301 0.295] players[avgs > .295] → ['Rivera' 'Walker']
Boolean indexing vs. np.where
np.where(condition, x, y) returns elements from array x where the condition is True, and from y where it is False. This is useful for conditional assignment: np.where(avgs > .300, 'hot', 'cold') returns ['hot', 'cold', 'hot', 'cold', 'cold'].

Fancy Indexing

Fancy indexing means passing an array of integers as an index - instead of a single index or a slice, you hand NumPy a list of positions and it returns exactly those elements, in that order. Unlike slicing, fancy indexing always returns a copy, not a view.

players = np.array(['Rivera', 'Chen', 'Walker', 'Kim', 'Jones'])

# Select specific indices in any order
idx = [0, 2, 4]
players[idx]     # ['Rivera', 'Walker', 'Jones']

# Repeat indices are allowed
players[[0, 0, 2]]  # ['Rivera', 'Rivera', 'Walker']

# Reverse order
players[[-1, -2, -3]]  # ['Jones', 'Kim', 'Walker']

# Fancy indexing in 2D: select specific rows
stats = np.array([
    [.312, 4, 22],
    [.287, 2, 15],
    [.301, 7, 31],
    [.268, 1, 9]
])
stats[[0, 2]]     # rows 0 and 2: Rivera and Walker
stats[[3, 1, 0]]  # any order you want

# Combine fancy + regular indexing
stats[[0, 2], 1]   # HRs for Rivera and Walker: [4, 7]
output — fancy indexing
players[[0,2,4]] → ['Rivera' 'Walker' 'Jones'] players[[0,0,2]] → ['Rivera' 'Rivera' 'Walker'] stats[[0,2]] → [[0.312 4. 22.] # Rivera [0.301 7. 31.]] # Walker stats[[0,2], 1] → [4. 7.] # HRs only

Sorting with argsort

avgs = np.array([.312, .287, .301, .268])

# argsort returns the indices that would sort the array
order = np.argsort(avgs)       # [3, 1, 2, 0] (ascending)

# Use those indices to sort both arrays together
players = np.array(['Rivera', 'Chen', 'Walker', 'Kim'])
players[order]  # ['Kim', 'Chen', 'Walker', 'Rivera']
avgs[order]     # [.268, .287, .301, .312]

# Descending: reverse the order array
players[order[::-1]]  # ['Rivera', 'Walker', 'Chen', 'Kim']
NumPy as pandas' engine
Every pandas Series and DataFrame column is backed by a NumPy array. When you call df['avg'].values, you get the underlying NumPy array. Understanding NumPy means you understand what pandas is doing under the hood - and you can drop down to NumPy when you need maximum performance.

Pseudorandom Number Generation

Random numbers are essential in data science for simulations, sampling, and setting up reproducible experiments. NumPy's random module generates pseudorandom numbers - they look random but are produced by a deterministic algorithm seeded with a starting value.

Setting a seed for reproducibility

A seed pins the random number generator so every run produces the same sequence. This is critical when you need reproducible results - for example, when splitting data for machine learning or writing tests.

import numpy as np

# Old-style (still common in tutorials)
np.random.seed(42)
np.random.randn(5)          # same 5 values every time

# Modern style (preferred)
rng = np.random.default_rng(42)
rng.normal(size=5)            # reproducible normal distribution
rng.integers(1, 7, size=10)  # 10 random integers 1-6 (like dice rolls)

Common random functions

rng = np.random.default_rng(0)

rng.random(5)          # 5 floats in [0, 1)
rng.normal(0, 1, 100) # 100 values from standard normal
rng.choice(students, size=2, replace=False)  # random partner pairs
rng.shuffle(arr)       # shuffle in place
Why "pseudo" random?
Computers can't generate true randomness - they use a mathematical formula that produces sequences that look random. The seed is just the starting point for that formula. Same seed = same sequence every time, which makes your analysis reproducible.

Conditional Logic as Array Operations

Instead of writing Python if/else loops over arrays (slow!), NumPy provides vectorized conditional operations that apply logic element-wise across entire arrays at once.

np.where - vectorized if/else

np.where(condition, x, y) returns elements from x where the condition is True, and elements from y where it is False.

scores = np.array([85, 72, 91, 60, 78])

# Label each score as Pass or Fail
labels = np.where(scores >= 75, 'Pass', 'Fail')
# ['Pass', 'Fail', 'Pass', 'Fail', 'Pass']

# Remap values: cap any score above 90 at exactly 90
capped = np.where(scores > 90, 90, scores)
# [85, 72, 90, 60, 78]

Replacing values in an array

Boolean indexing with assignment is another form of conditional logic - select elements that meet a condition, then replace them:

temps = np.array([55, 72, 89, 65, 91, 78])

# Replace any temperature above 85 with exactly 85
temps[temps > 85] = 85
# [55, 72, 85, 65, 85, 78]

# Categorize: hot vs cool
category = np.where(temps > 70, 'Hot', 'Cool')
np.where vs Python loops
A Python for loop checking 1 million values might take seconds. np.where on the same array takes milliseconds - it runs compiled C code under the hood. Always prefer vectorized operations when working with large datasets.

Concatenating, Stacking, and Splitting

Real data often arrives in pieces - sensor readings from different days, student scores from different exams, etc. NumPy provides tools to combine and split arrays efficiently.

Concatenation: joining 1D arrays

# Weather data from two weeks
week1_temps = np.array([72, 75, 68, 71, 74, 69, 73])
week2_temps = np.array([76, 78, 80, 77, 75, 72, 74])

two_weeks = np.concatenate([week1_temps, week2_temps])
# One 14-element array

Stacking: combining 2D arrays

np.vstack stacks arrays vertically (adds rows). np.hstack stacks horizontally (adds columns).

exam1 = np.array([[85, 90, 78],
                   [92, 88, 76]])

exam2 = np.array([[79, 84, 91],
                   [88, 93, 82]])

# vstack: 4 rows (both exams as separate rows)
np.vstack([exam1, exam2])   # shape (4, 3)

# hstack: add exam2 as new columns alongside exam1
np.hstack([exam1, exam2])   # shape (2, 6)

# vstack is equivalent to np.concatenate([...], axis=0)
# hstack is equivalent to np.concatenate([...], axis=1)

Splitting arrays

np.array_split divides an array into sub-arrays. Unlike np.split, it handles uneven splits without raising an error.

monthly_sales = np.array([120, 135, 150,   # Q1
                           180, 165, 200,   # Q2
                           210, 195, 220,   # Q3
                           240, 260, 280])  # Q4

# Split evenly into 4 quarters
q1, q2, q3, q4 = np.split(monthly_sales, 4)
# q1 = [120, 135, 150]

# Split at specific indices
first_half, second_half = np.split(monthly_sales, [6])
# first_half = first 6 months, second_half = last 6
vstack vs concatenate(axis=0)
np.vstack and np.concatenate([...], axis=0) do the same thing. vstack is more readable when you're stacking rows; concatenate is more explicit and flexible. You'll see both in practice - knowing they're equivalent prevents confusion.
