Series: The 1D Building Block

A Series is pandas' one-dimensional labeled array. Think of it as a single column from a spreadsheet - every value has an index label alongside it. Series are the atoms from which DataFrames are built.

import pandas as pd

# Create a Series from a list
batting_avgs = pd.Series([.312, .287, .301, .268],
                         index=['Rivera', 'Chen', 'Walker', 'Kim'])

# Access by label
print(batting_avgs['Rivera'])   # 0.312

# Vectorized math - no loop needed
slugging = batting_avgs * 1.4
output — batting_avgs Series
Rivera 0.312 Chen 0.287 Walker 0.301 Kim 0.268 dtype: float64
output — slugging = batting_avgs * 1.4
Rivera 0.4368 Chen 0.4018 Walker 0.4214 Kim 0.3752 dtype: float64

DataFrames: Tabular Data

A DataFrame is a 2-dimensional labeled table - rows and columns, like a spreadsheet or SQL table. Each column is a Series, all sharing the same index. This is the central data structure for nearly all pandas work.

# Create a DataFrame from a dict
data = {
    'player': ['Rivera', 'Chen', 'Walker', 'Kim'],
    'avg':    [.312, .287, .301, .268],
    'hr':     [4, 2, 7, 1],
    'salary': [16800, 14400, 19200, 760000]  # Kim on MLB rehab
}
roster = pd.DataFrame(data)

# Quick inspection
roster.head()          # first 5 rows
roster.info()          # column types + null count
roster.describe()      # numeric summary stats
roster.shape           # (4, 4)
roster.columns         # Index(['player','avg','hr','salary'])
output — roster.head()
playeravghrsalary
0Rivera0.312416800
1Chen0.287214400
2Walker0.301719200
3Kim0.2681760000
shape: (4, 4)    dtypes: object(1), float64(1), int64(2)
Real-world loading
Most of the time you won't build DataFrames manually - you'll load them from files. The two most common entry points are pd.read_csv('file.csv') and pd.read_excel('file.xlsx'). Both return a DataFrame ready to work with. 
# Load CSV with common options
roster = pd.read_csv(
    'eagles_roster.csv',
    index_col='player_id',  # use this column as row index
    parse_dates=['dob'],     # convert string dates
    na_values=['N/A', '-']  # treat these as missing
)

Selection: Accessing Your Data

Pandas gives you multiple ways to pull out rows, columns, or individual cells. The two main tools are .loc[] (label-based) and .iloc[] (integer position-based). Getting these right is one of the most important pandas fundamentals.

.loc[ ] - Label-based

Uses actual index labels and column names. Inclusive on both ends of slices.

df.loc[0, 'avg'] - single cell

df.loc[:, 'avg'] - entire column

df.loc[0:2, 'avg':'hr'] - slice of rows and cols

.iloc[ ] - Position-based

Uses integer positions (0-indexed). Exclusive on the end of slices, like Python lists.

df.iloc[0, 1] - row 0, column 1

df.iloc[:, 2] - entire 3rd column

df.iloc[0:2, 1:3] - rows 0-1, cols 1-2

# Single column - returns a Series
roster['avg']

# Multiple columns - returns a DataFrame
roster[['player', 'avg', 'hr']]

# loc: label-based
roster.loc[0, 'avg']              # single value: 0.312
roster.loc[0:2, ['player', 'avg']] # rows 0-2, two columns

# iloc: integer position
roster.iloc[0, 1]    # row 0, col 1
roster.iloc[:2]     # first two rows
output — roster[['player','avg','hr']]
playeravghr
0Rivera0.3124
1Chen0.2872
2Walker0.3017
3Kim0.2681
roster.loc[0, 'avg'] → 0.312
Common mistake
Chaining selections like df['col'][0] can cause a SettingWithCopyWarning and unpredictable behavior when assigning values. Always use .loc[] or .iloc[] for assignment: df.loc[0, 'col'] = value.

Filtering: Selecting Rows by Condition

Filtering uses a boolean mask - a Series of True/False values aligned to the DataFrame's index. Rows where the mask is True are kept; rows where it's False are dropped.

# Boolean comparison creates the mask
roster[roster['avg'] > .300]

# Combine conditions with & (and), | (or), ~ (not)
# IMPORTANT: wrap each condition in parentheses
roster[(roster['avg'] > .280) & (roster['hr'] >= 4)]

# Filter using a list of values
roster[roster['player'].isin(['Rivera', 'Walker'])]

# .query() - readable string syntax
roster.query("avg > .290 and hr > 2")
output — roster[roster['avg'] > .300]
playeravghrsalary
0Rivera0.312416800
2Walker0.301719200
# .query("avg > .290 and hr > 2") →
playeravghrsalary
0Rivera0.312416800
2Walker0.301719200

Filtering missing values

# Keep rows where 'avg' is NOT missing
roster[roster['avg'].notna()]

# Drop rows with any missing value
roster.dropna()

# Drop only when specific columns are missing
roster.dropna(subset=['avg', 'hr'])

Data Cleaning

Real datasets are messy. Missing values, wrong types, duplicates, and inconsistent strings are the norm. Pandas has a rich toolkit for identifying and fixing these issues before analysis.

Handling missing values

# Detect missing values
roster.isna().sum()   # count per column

# Fill missing with a constant
roster['hr'].fillna(0)

# Fill with column mean (imputation)
roster['avg'].fillna(roster['avg'].mean())

# Drop rows with missing values in key columns
roster_clean = roster.dropna(subset=['avg', 'salary'])

Fixing data types

# Column came in as string, should be numeric
roster['salary'] = (
    roster['salary']
    .str.replace('$', '', regex=False)
    .str.replace(',', '', regex=False)
    .astype(float)
)

# Convert to datetime
roster['signed_date'] = pd.to_datetime(roster['signed_date'])

Removing duplicates and cleaning strings

# Find and remove duplicate rows
roster.drop_duplicates(subset=['player'])

# String cleaning
roster['player'].str.strip()   # remove leading/trailing whitespace
roster['player'].str.lower()   # normalize to lowercase

# Rename columns
roster.rename(columns={'avg': 'batting_avg', 'hr': 'home_runs'})
In-place vs. returning new objects
Most pandas cleaning methods return a new DataFrame by default rather than modifying the original. To make changes permanent, either reassign (df = df.dropna()) or use inplace=True. Reassignment is generally preferred as it is clearer and easier to debug.

Aggregation: Summarizing Data

Aggregation collapses many rows into summary statistics. Pandas provides both column-level methods and the flexible .agg() interface for computing multiple statistics at once.

# Single-column aggregations
roster['salary'].mean()    # average salary
roster['salary'].median()  # median (outlier-resistant)
roster['salary'].max()     # 760000 (Kim)
roster['salary'].sum()     # total payroll

# Multiple stats at once with .agg()
roster['salary'].agg(['mean', 'median', 'std', 'max'])

# Different stats per column
roster.agg({
    'salary': ['mean', 'max'],
    'avg':    ['mean', 'min', 'max'],
    'hr':     'sum'
})
output — roster['salary'].agg(['mean','median','std','max'])
mean 202,600.00 median 18,000.00 std 372,284.17 max 760,000.00 Name: salary, dtype: float64
Why median matters for salary data
The Eagles roster has salaries ranging from $14,400 to $760,000. The mean salary (~$202,600) is misleading - it is inflated by Kim's MLB rehab assignment. The median (~$18,000) better represents what a typical player earns. Whenever you have potential outliers, always compare mean vs. median before reporting a "typical" value.

GroupBy: Split-Apply-Combine

GroupBy divides your data into groups, applies a function to each group, and combines the results. This "split-apply-combine" pattern is one of the most powerful tools for exploratory analysis.

# Add a position column for demonstration
roster['position'] = ['OF', 'IF', 'OF', 'IF']

# Mean batting avg by position
roster.groupby('position')['avg'].mean()

# Multiple aggregations per group
roster.groupby('position').agg({
    'avg':    ['mean', 'max'],
    'salary': 'mean',
    'hr':     'sum'
})

# Custom function per group
def above_avg_pct(series):
    return (series > series.mean()).mean() * 100

roster.groupby('position')['avg'].apply(above_avg_pct)

Sorting and ranking

# Sort by batting average, descending
roster.sort_values('avg', ascending=False)

# Get top 2 by avg
roster.nlargest(2, 'avg')

# Add a rank column
roster['avg_rank'] = roster['avg'].rank(ascending=False)

Merging DataFrames
When data is split across multiple tables (like player stats and contracts), use pd.merge() to join them - similar to SQL JOIN. The most common form: pd.merge(df1, df2, on='player_id', how='left'). The how parameter accepts 'inner', 'left', 'right', and 'outer', matching SQL join types.

Index Objects

Every pandas Series and DataFrame has an Index - the row labels. The Index is an immutable array-like object that enables fast lookup, alignment, and slicing. Understanding it helps explain pandas behavior that might otherwise seem surprising.

import pandas as pd

players = pd.Series([.312, .287, .301],
                   index=['Rivera', 'Chen', 'Walker'])

# Access index information
players.index           # Index(['Rivera', 'Chen', 'Walker'])
players.index.tolist() # ['Rivera', 'Chen', 'Walker']
'Rivera' in players.index  # True

# Reindexing: conform data to a new set of labels
# Missing labels become NaN by default
full_roster = ['Rivera', 'Chen', 'Walker', 'Kim']
players.reindex(full_roster)
# Kim gets NaN since she wasn't in the original

# Fill missing values when reindexing
players.reindex(full_roster, fill_value=0.000)
# Reset index on a DataFrame (after filtering, the original
# row numbers are preserved - reset gives you 0, 1, 2, ...)
starters = roster[roster['ab'] > 300].reset_index(drop=True)

# Set a column as the index
roster = roster.set_index('player_id')
roster.loc['P042']  # look up by player ID directly

Arithmetic and Data Alignment

When you perform arithmetic between two pandas objects, pandas automatically aligns on the index before computing. This prevents silent errors when data comes from different sources with different row orders - but it also introduces NaN when labels don't match.

week1 = pd.Series([200, 150, 300], index=['Mon', 'Tue', 'Wed'])
week2 = pd.Series([180, 210, 275], index=['Mon', 'Tue', 'Thu'])

# Wed and Thu only appear in one series - they become NaN
week1 + week2
# Mon  380.0, Tue  360.0, Wed  NaN, Thu  NaN

# Use fill_value to treat missing as 0 instead
week1.add(week2, fill_value=0)
# Mon  380, Tue  360, Wed  300, Thu  210
# Arithmetic between a DataFrame and a Series
# The Series index aligns with the DataFrame COLUMNS by default
df - df.mean()   # subtract each column's mean from every row

# Sort a DataFrame by index label
roster.sort_index()             # ascending alphabetical
roster.sort_index(ascending=False)  # descending
Why alignment matters
Imagine you have two DataFrames from different data pulls - same players but in different row orders. Without index alignment, a naive NumPy subtraction would compare wrong rows. Pandas fixes this by aligning on the index first, so you always get a meaningful result (or an explicit NaN warning you something didn't match).

Summarizing and Descriptive Statistics

Pandas provides reduction methods that compute a summary statistic across rows or columns. These are essential for getting a quick feel for your data.

# .describe() gives a full statistical summary at once
roster.describe()
# count, mean, std, min, 25%, 50%, 75%, max for every numeric column

# Individual statistics
roster['avg'].mean()    # mean batting average
roster['avg'].median()  # median
roster['avg'].std()     # standard deviation
roster['hr'].sum()      # total home runs
roster['hr'].max()      # max home runs
roster['hr'].idxmax()  # INDEX of the row with max home runs

# Apply across rows or columns with the axis argument
# axis=0 (default): compute down each column
# axis=1: compute across each row
roster[['hr', 'rbi']].sum(axis=1)  # total production per player

Unique Values, Value Counts, and Membership

These methods are building blocks for exploratory data analysis - quickly understanding the range and distribution of categorical and discrete data.

# unique(): what distinct values exist?
roster['position'].unique()
# array(['OF', 'IF', 'C', 'P'], dtype=object)

# value_counts(): how often does each value appear?
roster['position'].value_counts()
# OF    8
# IF    7
# P     5
# C     2

# Normalize to get proportions instead of counts
roster['position'].value_counts(normalize=True)

# isin(): filter rows where a column's value is in a list
corner_inf = roster[roster['position'].isin(['1B', '3B'])]

# Useful for exclusion too (using ~)
non_pitchers = roster[~roster['position'].isin(['P', 'RP'])]
output — roster['position'].value_counts()
OF 2 IF 2 Name: position, dtype: int64
# .value_counts(normalize=True)
OF 0.50 IF 0.50 Name: position, dtype: float64
Checking for unexpected categories
value_counts() is often the first thing to run on a categorical column after loading data. It reveals typos ('Outfield' vs 'OF'), encoding inconsistencies ('M' vs 'Male'), and unexpected values that would break downstream analysis.

MultiIndex: Hierarchical Indexing

A MultiIndex (also called a hierarchical index) lets you work with data that has multiple levels of grouping - for example, performance stats organized by both team and player, or sales data by year and quarter.

arrays = [
    ['Eagles', 'Eagles', 'Falcons', 'Falcons'],
    ['Rivera', 'Chen',   'Lopez',   'Smith']
]
index = pd.MultiIndex.from_arrays(arrays,
                                    names=['team', 'player'])
stats = pd.DataFrame({'avg': [.312, .287, .301, .265]},
                     index=index)

# Select all Eagles
stats.loc['Eagles']

# Select a specific player
stats.loc[('Eagles', 'Rivera')]

# Flatten back to a regular index
stats.reset_index()
When to use MultiIndex
MultiIndex shines when data is naturally hierarchical and you need to slice at different levels (all data for a team, then drill down to a player). For simpler analysis, groupby() is often easier. You'll encounter MultiIndex in output from groupby operations, so knowing how to navigate it is important even if you rarely create it manually.

Combining Data: concat, join, and merge

Real data pipelines assemble information from multiple sources. Pandas offers three main tools - each suited to a different combining pattern.

concat - stacking DataFrames

Use pd.concat() when you have the same columns across multiple DataFrames and want to stack them vertically (add rows), or the same rows with different columns (add columns).

fall_enrollment = pd.DataFrame({'student': ['Ana', 'Bo'], 'course': ['DTSC-520', 'DTSC-560']})
spring_enrollment = pd.DataFrame({'student': ['Cal', 'Dev'], 'course': ['DTSC-520', 'DTSC-575']})

all_enrollment = pd.concat([fall_enrollment, spring_enrollment],
                            ignore_index=True)

merge - SQL-style joins

Use pd.merge() when two DataFrames share a key column and you want to combine their columns - equivalent to a SQL JOIN.

students = pd.DataFrame({'student_id': [1, 2, 3],
                          'name': ['Ana', 'Bo', 'Cal']})
gpa = pd.DataFrame({'student_id': [1, 2, 4],
                     'gpa': [3.8, 3.5, 3.9]})

# inner join: only students in both DataFrames
pd.merge(students, gpa, on='student_id', how='inner')

# left join: all students, NaN gpa if not in gpa df
pd.merge(students, gpa, on='student_id', how='left')

join - index-based combining

df.join() is a shorthand for merging on the index rather than a column.

program_info = pd.DataFrame({'program': ['MSDS', 'MBA']},
                             index=[1, 2])
students.set_index('student_id').join(program_info)
concat vs merge
Use concat when tables have the same structure and you're stacking them (more rows or more columns). Use merge when tables have different columns linked by a shared key (like a student ID). When in doubt, merge is the more powerful and flexible tool.

Pivot Tables

A pivot table reorganizes data by grouping on one or more columns and applying an aggregation - think of it as a more flexible, readable alternative to groupby() for multi-dimensional summaries.

# Sample sales data
sales = pd.DataFrame({
    'region': ['East', 'West', 'East', 'West'],
    'product': ['Laptop', 'Laptop', 'Phone', 'Phone'],
    'revenue': [50000, 45000, 30000, 35000]
})

# Create a pivot table: rows=region, columns=product, values=revenue
pd.pivot_table(sales,
               values='revenue',
               index='region',
               columns='product',
               aggfunc='sum')

# Output:
# product   Laptop   Phone
# region
# East       50000   30000
# West       45000   35000

# Multiple aggregations at once
pd.pivot_table(sales, values='revenue', index='region',
               aggfunc=['sum', 'mean', 'count'])
output — pivot_table(values='revenue', index='region', columns='product', aggfunc='sum')
regionproduct
LaptopPhone
East5000030000
West4500035000
groupby vs pivot_table
Both do split-apply-combine, but pivot_table produces a 2D grid that's easier to read when you have two grouping dimensions. For a single grouping column, groupby() is simpler. When you need to cross-tabulate two categorical columns (region x product, month x category), reach for pivot_table.
