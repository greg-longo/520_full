01 Getting Started

Python programs run line by line, top to bottom. The most basic program you can write is a single print() call - it displays output to the screen.

print("Hello, world!")
print("StatDeck Analytics - ready.")

Python as a Calculator

Python handles arithmetic directly. Operators work as you'd expect, with one exception worth noting: division always returns a float.

# Basic arithmetic
print(10 + 3)      # 13
print(10 - 3)      # 7
print(10 * 3)      # 30
print(10 / 3)      # 3.3333... (always float)
print(10 // 3)     # 3 (integer division)
print(10 % 3)      # 1 (remainder)
print(2 ** 8)      # 256 (exponent)
output
13 7 30 3.3333333333333335 3 1 256

02 Variables & Data Types

A variable is a named container for a value. You create one with the assignment operator =. Python figures out the type automatically.

# Assigning variables
player_name = "Marcus Rivera"     # str
jersey_number = 14               # int
batting_avg = 0.287              # float
is_eligible = True               # bool

print(player_name, "#" + str(jersey_number))
print(f"Batting average: {batting_avg}")
output
Marcus Rivera #14 Batting average: 0.287

int

Whole numbers. 14, -3, 0. No decimal point.

float

Decimal numbers. 0.287, 3.14, -1.5.

str

Text in quotes. "Marcus", 'StatDeck'. Immutable.

bool

Either True or False. Used in conditions.

Type Conversion

Types don't automatically mix. Use int(), float(), and str() to convert between them.

age_str = "25"
age_int = int(age_str)         # "25" → 25

games = 82
label = "Games: " + str(games) # must convert int to str first

print(type(batting_avg))       # <class 'float'>

Augmented Assignment Operators

These operators update a variable in place - a shorter way to write common update patterns.

hits = 45
hits += 3     # hits = hits + 3  →  48
hits -= 1     # hits = hits - 1  →  47
hits *= 2     # hits = hits * 2  →  94
hits //= 5    # hits = hits // 5  →  18

# Commonly used in loops to keep a running total
total_runs = 0
for game_runs in [3, 1, 5, 2]:
    total_runs += game_runs
print(total_runs)   # 11

Variable Names and Keywords

Variable names must start with a letter or underscore, can contain letters, digits, and underscores, and are case-sensitive. Python uses snake_case by convention (batting_avg, not BattingAvg).

Certain words are reserved keywords and cannot be used as variable names:

# Python keywords - cannot be used as variable names
# False  None   True   and    as     assert  async  await
# break  class  continue  def  del    elif   else   except
# finally  for  from  global  if  import  in  is  lambda
# nonlocal  not  or  pass  raise  return  try  while  with  yield

# These will cause SyntaxError:
# class = "Freshman"   ← 'class' is a keyword
# return = 42          ← 'return' is a keyword

# Valid alternatives:
student_class = "Freshman"
return_value = 42

Comparison Operators

Comparisons return a boolean (True or False) and are the foundation of all decision-making in Python.

batting_avg = 0.287
print(batting_avg > 0.300)    # False
print(batting_avg >= 0.250)   # True
print(batting_avg == 0.287)   # True  (note: == not =)
print(batting_avg != 0.300)   # True

03 Strings

Strings are sequences of characters. You can use single quotes, double quotes, or triple quotes (for multi-line strings). Strings are immutable - you can't change individual characters, but you can build new strings from old ones.

name = "St. Davids Eagles"
print(name.upper())          # ST. DAVIDS EAGLES
print(name.lower())          # st. davids eagles
print(name.replace("Eagles", "A's"))
print(name.split("."))        # ['St', ' Davids Eagles']
print(len(name))              # 17
output
ST. DAVIDS EAGLES st. davids eagles St. Davids A's ['St', ' Davids Eagles'] 17

Indexing & Slicing

Every character in a string has a position (index), starting at 0. Slicing lets you extract a substring using [start:stop].

team = "Eagles"
print(team[0])       # E  (first character)
print(team[-1])      # s  (last character)
print(team[0:3])     # Eag  (index 0,1,2 - stop is exclusive)
print(team[:3])      # Eag  (same - start defaults to 0)

f-strings

f-strings (formatted string literals) are the cleanest way to embed variables inside text. Prefix with f and put variables in {}.

name = "Marcus Rivera"
avg = 0.287
print(f"{name} is batting {avg:.3f} this season.")
# Marcus Rivera is batting 0.287 this season.
output — string slicing + f-string
team[0] → 'E' team[-1] → 's' team[0:3] → 'Eag' Marcus Rivera is batting 0.287 this season.

04 Conditions

Conditions let your program make decisions. The if statement checks a boolean expression; if it's True, the indented block runs. Python uses indentation (4 spaces) to define blocks - not curly braces.

batting_avg = 0.287

if batting_avg >= 0.300:
    print("All-Star candidate")
elif batting_avg >= 0.260:
    print("Solid starter")      # this runs
else:
    print("Needs improvement")
output — batting_avg = 0.287
Solid starter

Boolean Operators

Combine conditions with and, or, and not.

avg = 0.287
games = 120

if avg >= 0.260 and games >= 100:
    print("Qualifies for batting title")

if "Rivera" in player_name:
    print("Found Rivera on roster")

if not is_eligible:
    print("Player is ineligible")
= vs ==

= assigns a value to a variable. == compares two values. Using = inside an if condition is one of the most common beginner mistakes.

05 Loops

Loops let you repeat code without copy-pasting. Python has two kinds: for loops (iterate over a sequence) and while loops (repeat while a condition is true).

for loops

roster = ["Rivera", "Chen", "Okafor", "Patel"]

for player in roster:
    print(f"Player: {player}")

# range() generates a sequence of numbers
for i in range(5):
    print(i)           # 0, 1, 2, 3, 4

# enumerate() gives index AND value
for i, player in enumerate(roster):
    print(f"{i}: {player}")
output — for loop + enumerate
Player: Rivera Player: Chen Player: Okafor Player: Patel --- enumerate --- 0: Rivera 1: Chen 2: Okafor 3: Patel

while loops

score = 0
while score < 3:
    print(f"Score is {score}")
    score += 1        # don't forget to update - or loop forever
output — while loop
Score is 0 Score is 1 Score is 2

break & continue

for player in roster:
    if player == "Okafor":
        break           # stop the loop entirely
    print(player)

for player in roster:
    if player == "Chen":
        continue        # skip this iteration, keep going
    print(player)
output — break (stops at Okafor) / continue (skips Chen)
--- break --- Rivera Chen --- continue --- Rivera Okafor Patel

06 Built-In Data Structures

Python's built-in data structures let you store collections of values. Choosing the right one for the job matters.

list

Ordered, mutable. Use for sequences you'll modify. [1, 2, 3]

tuple

Ordered, immutable. Use for fixed data. (1, 2, 3)

dict

Key-value pairs, mutable. Use for labeled data. {"avg": 0.287}

set

Unordered, unique values. Use to deduplicate. {1, 2, 3}

Lists

roster = ["Rivera", "Chen", "Okafor"]

roster.append("Patel")        # add to end
roster.insert(0, "Garcia")   # add at index
roster.remove("Chen")        # remove by value
print(roster[0])              # first element
print(len(roster))            # count
print("Rivera" in roster)     # True
output — list after append/insert/remove
roster → ['Garcia', 'Rivera', 'Okafor', 'Patel'] roster[0] → 'Garcia' len(roster) → 4 'Rivera' in roster → True

Dictionaries

player = {
    "name": "Marcus Rivera",
    "number": 14,
    "avg": 0.287,
    "stats": {"HR": 8, "RBI": 42}   # nested dict
}

print(player["name"])          # Marcus Rivera
print(player["stats"]["HR"])  # 8
player["avg"] = 0.301          # update a value

for key, val in player.items():
    print(f"{key}: {val}")
output — dictionary access + .items() loop
player["name"] → 'Marcus Rivera' player["stats"]["HR"] → 8 --- .items() loop --- name: Marcus Rivera number: 14 avg: 0.301 stats: {'HR': 8, 'RBI': 42}

07 Functions

A function is a reusable block of code with a name. You define it once with def, then call it as many times as you need. Functions make code shorter, more readable, and easier to debug.

def greet_player(name):
    print(f"Welcome to StatDeck, {name}!")

greet_player("Marcus Rivera")   # call it
greet_player("Jonah Reyes")
output
Welcome to StatDeck, Marcus Rivera! Welcome to StatDeck, Jonah Reyes!

Parameters & Return Values

Functions can take multiple parameters and return a value with return. Default parameter values make some arguments optional.

def batting_grade(avg, threshold=0.270):
    if avg >= threshold + 0.030:
        return "Excellent"
    elif avg >= threshold:
        return "Solid"
    else:
        return "Needs work"

grade = batting_grade(0.287)
print(grade)                   # Solid
print(batting_grade(0.315))   # Excellent
output — batting_grade(0.287) / batting_grade(0.315)
Solid Excellent

return vs print

print() displays output to the screen but the value disappears. return sends a value back to the caller so it can be stored and used. In data science, you almost always want return.

def double_print(x):
    print(x * 2)       # works once, then gone

def double_return(x):
    return x * 2       # caller gets the value

result = double_return(5)
print(result + 1)    # 11 - we can use the returned value

Comments

A comment is text in your code that Python ignores. Use comments to explain why your code does something, not just what it does. Good comments make code understandable months later - or to a teammate reading it for the first time.

# This is a single-line comment
batting_avg = 0.287   # inline comment - use sparingly

# Avoid redundant comments (this one says nothing new):
# x = x + 1   # add 1 to x

# Useful comment - explains why, not just what:
# Multiply by 1000 to convert from feet to inches (sensor outputs feet)
height_in = sensor_value * 12

Variable Scope

Scope determines where a variable is accessible. Variables defined inside a function are local - they exist only while the function runs and cannot be accessed outside it.

def calc_ops(ab, hits):
    avg = hits / ab          # 'avg' is local to this function
    return avg

result = calc_ops(400, 115)
print(result)   # 0.2875 - OK

# print(avg)    # NameError: 'avg' not defined here
# Variables defined outside functions are 'global'
# and accessible everywhere - but modifying globals inside
# functions is usually a design smell to avoid
