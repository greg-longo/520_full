Matplotlib & Seaborn: Communicating with Charts

Choosing the Right Chart

The most common visualization mistake is choosing a chart type based on aesthetics rather than the structure of your data and the question you are answering. Every chart type has a job. Using the wrong one obscures the truth; using the right one makes it obvious.

Line Chart

Use when: showing change over time or continuous data.

Emphasizes trends and continuity. Ideal for time series - batting average over a season, daily ticket sales. The connected line implies the values between points exist and matter.

Bar Chart

Use when: comparing discrete categories.

Each bar represents one category. The eye compares heights (or lengths in a horizontal bar). Good for: HR counts by player, revenue by region. Always start the y-axis at zero.

Scatter Plot

Use when: showing relationship between two continuous variables.

Each point is one observation. Reveals correlation, clusters, and outliers. Classic use: batting average vs. on-base percentage. Add color or size for a third variable.

Histogram

Use when: showing distribution of a single variable.

Groups continuous values into bins and counts how many observations fall in each bin. Shows skew, modality, and spread. Essential for EDA - always plot your distributions before modeling.

Box Plot

Use when: comparing distributions across categories.

Shows median, quartiles, and outliers in a compact form. Excellent for "salary distribution by position" or "batting avg distribution by team." More informative than a mean-only bar chart.

Heatmap

Use when: showing values in a matrix (correlations, co-occurrences).

Encodes values as color intensity. Most common use: correlation matrix of numeric features. Reveals which variables move together - essential before building any model.

Charts to avoid
Pie charts are rarely the right choice - humans are poor at comparing angles. Use a bar chart instead. 3D charts almost always distort perception. Dual-axis charts often imply a relationship that does not exist. When in doubt, simpler is better.

Matplotlib: The Foundation

Matplotlib is the core plotting library for Python. Almost every other visualization library - Seaborn, pandas' built-in plotting, and others - is built on top of it. Understanding Matplotlib's architecture makes you effective with all of them.

The Figure and Axes model

Matplotlib has two key objects. A Figure is the entire canvas - the window or image file. An Axes is one plot within that canvas (despite the name, it is not just the axis lines). A Figure can contain multiple Axes (subplots). Most of your configuration happens on the Axes object.

import matplotlib.pyplot as plt
import numpy as np

# Data: Eagles roster batting averages over 10 weeks
weeks = np.arange(1, 11)
rivera = np.array([.290, .295, .300, .305, .308, .310, .309, .311, .312, .314])
walker  = np.array([.275, .280, .288, .291, .295, .299, .300, .301, .301, .303])

# Object-oriented interface (preferred)
fig, ax = plt.subplots(figsize=(8, 4))

ax.plot(weeks, rivera, label='Rivera', color='#cc5595', linewidth=2)
ax.plot(weeks, walker,  label='Walker',  color='#6baacc', linewidth=2)

ax.set_title('Batting Average by Week', fontsize=14)
ax.set_xlabel('Week')
ax.set_ylabel('Batting Average')
ax.legend()
ax.grid(True, alpha=0.3)

plt.tight_layout()
plt.savefig('batting_trend.png', dpi=150)
plt.show()
Rendered output - Line Chart: Batting Average by Week

Common plot types in Matplotlib

fig, ax = plt.subplots()

# Bar chart
players = ['Rivera', 'Chen', 'Walker', 'Kim']
hrs      = [4, 2, 7, 1]
ax.bar(players, hrs, color='#3a8fa0', alpha=0.85)
ax.set_ylabel('Home Runs')

# Scatter plot
avgs     = [.312, .287, .301, .268]
salaries = [16800, 14400, 19200, 760000]
ax.scatter(avgs, salaries, s=80, color='#3a8fa0', alpha=0.7)

# Histogram
data = np.random.normal(.290, .020, 200)  # simulated batting avgs
ax.hist(data, bins=20, color='#3a8fa0', edgecolor='black', alpha=0.75)

# Horizontal bar (easier to read long labels)
ax.barh(players, hrs, color='#3a8fa0')

Seaborn: Statistical Visualization

Seaborn is built on top of Matplotlib and integrates tightly with pandas DataFrames. It makes statistical plots that would require significant Matplotlib code just a single function call. It also applies attractive default styles automatically.

import seaborn as sns
import pandas as pd

roster = pd.DataFrame({
    'player':   ['Rivera', 'Chen', 'Walker', 'Kim', 'Jones', 'Patel'],
    'avg':      [.312, .287, .301, .268, .295, .310],
    'hr':       [4, 2, 7, 1, 3, 5],
    'position': ['OF', 'IF', 'OF', 'IF', 'OF', 'IF']
})

# Scatter with hue - color by category
sns.scatterplot(data=roster, x='avg', y='hr', hue='position', s=100)

# Box plot - distribution by group
sns.boxplot(data=roster, x='position', y='avg')

# Bar plot with automatic error bars
sns.barplot(data=roster, x='position', y='avg')

# Distribution plot (histogram + KDE)
sns.histplot(data=roster, x='avg', kde=True)
Seaborn Scatter - Avg vs HR, colored by Position
Seaborn Histogram + KDE - Batting Average Distribution
Seaborn vs. Matplotlib
Use Seaborn when you want quick, attractive statistical charts that work directly with pandas DataFrames. Use Matplotlib when you need fine-grained control over layout, annotations, or custom visual elements. In practice, you will often use both together - Seaborn to draw the plot, then Matplotlib methods (ax.set_title(), ax.set_xlabel()) to customize it.

Subplots: Multiple Charts in One Figure

Subplots let you arrange multiple related charts side by side for comparison. The key function is plt.subplots(nrows, ncols), which returns a Figure and an array of Axes objects.

# 1 row, 2 columns
fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(12, 4))

# Left: batting averages
ax1.bar(players, avgs, color='#3a8fa0', alpha=0.85)
ax1.set_title('Batting Averages')
ax1.set_ylabel('Average')
ax1.set_ylim(0, 0.4)

# Right: home run counts
ax2.bar(players, hrs, color='#6baacc', alpha=0.85)
ax2.set_title('Home Runs')
ax2.set_ylabel('Home Runs')

fig.suptitle('Eagles Roster - Key Stats', fontsize=14, y=1.02)
plt.tight_layout()
plt.show()

# 2x2 grid
fig, axes = plt.subplots(2, 2, figsize=(10, 8))

# Access individual axes
axes[0, 0].plot(weeks, rivera)   # top-left
axes[0, 1].hist(data)            # top-right
axes[1, 0].bar(players, hrs)      # bottom-left
axes[1, 1].scatter(avgs, hrs)    # bottom-right

plt.tight_layout()  # prevents overlap
Rendered output - 2x2 Subplot Grid: Eagles Roster Stats

Styling Your Charts

Default Matplotlib charts are functional but plain. A few small changes - a clean style, consistent colors, clear labels - dramatically improve readability and professionalism.

Built-in styles

# Apply a built-in style globally
plt.style.use('seaborn-v0_8-darkgrid')  # clean grid
plt.style.use('ggplot')                # R-inspired style
plt.style.use('dark_background')        # dark theme

# See all available styles
print(plt.style.available)

Key customizations

fig, ax = plt.subplots(figsize=(8, 5))

ax.bar(players, avgs, color='#3a8fa0', edgecolor='white', linewidth=0.5)

# Labels and title
ax.set_title('Batting Averages - Eagles Roster', fontsize=14, pad=12)
ax.set_xlabel('Player', fontsize=11)
ax.set_ylabel('Batting Average', fontsize=11)

# Axis limits and tick formatting
ax.set_ylim(0, 0.40)
ax.yaxis.set_major_formatter(plt.FuncFormatter(lambda y, _: f'{y:.3f}'))

# Remove unnecessary spines (borders)
ax.spines['top'].set_visible(False)
ax.spines['right'].set_visible(False)

# Add value labels on bars
for bar in ax.patches:
    ax.text(
        bar.get_x() + bar.get_width() / 2,
        bar.get_height() + 0.003,
        f'{bar.get_height():.3f}',
        ha='center', va='bottom', fontsize=9
    )

plt.tight_layout()

Plotting with Pandas

Pandas DataFrames and Series have a built-in .plot() method that wraps Matplotlib. For quick, exploratory plots, df.plot() is faster to write than calling Matplotlib directly - it automatically uses the DataFrame's column names as labels and the index as the x-axis.

import pandas as pd
import matplotlib.pyplot as plt

monthly = pd.DataFrame({
    'revenue': [45000, 52000, 48000, 61000],
    'expenses': [38000, 41000, 39000, 44000]
}, index=['Q1', 'Q2', 'Q3', 'Q4'])

# Line plot (default)
monthly.plot()
plt.title('Revenue vs Expenses')
plt.ylabel('USD')
plt.show()

Common .plot() chart types

# Bar chart
monthly.plot(kind='bar')

# Horizontal bar
monthly.plot(kind='barh')

# Histogram of a single column
monthly['revenue'].plot(kind='hist', bins=10)

# Area chart (stacked by default)
monthly.plot(kind='area', alpha=0.5)

# Scatter plot (requires x and y columns)
monthly.plot(kind='scatter', x='revenue', y='expenses')
df.plot(kind='bar') - Revenue vs Expenses by Quarter
df.plot(kind='area') - Stacked Area Chart
df.plot() vs raw Matplotlib
df.plot() is great for quick exploratory analysis during data cleaning and EDA. It produces a Matplotlib Axes object that you can customize further using regular Matplotlib commands. For publication-quality or highly customized charts, work directly with Matplotlib's fig, ax = plt.subplots() pattern - but don't hesitate to start with df.plot() to see what the data looks like first.

Annotations: Adding Text and Reference Lines

Annotations guide the reader to the most important parts of a chart. They include text labels, arrows pointing to data points, and reference lines marking thresholds or averages.

Adding text and annotations

fig, ax = plt.subplots()
ax.bar(['Rivera', 'Chen', 'Walker'], [.312, .287, .301])

# Add text at a specific location (x, y)
ax.text(0, .315, 'Team best', ha='center', fontsize=9)

# Annotate with an arrow pointing to a data point
ax.annotate('Slump recovery',
            xy=(1, .287),            # point the arrow at
            xytext=(1.3, .295),      # where the label goes
            arrowprops=dict(arrowstyle='->'))

Reference lines

# Horizontal reference line (e.g., league average)
ax.axhline(y=.285, color='gray', linestyle='--',
           label='League avg')

# Vertical reference line (e.g., trade deadline)
ax.axvline(x='Walker', color='red', linestyle=':')

ax.legend()
plt.show()
Rendered output - Bar Chart with Annotations and Reference Line
When to annotate
Don't annotate everything - only the data points that need context the chart doesn't already provide. A good annotation answers a question the reader is likely to ask: "Why does this spike?" or "What threshold are we comparing against?" Use sparingly so annotations stay meaningful.

Visualization Best Practices

A technically correct chart can still mislead. These principles come from data visualization research and decades of practitioner experience. Apply them every time.

  • 01 Start the y-axis at zero for bar charts. Truncating the y-axis exaggerates differences. A bar chart with a y-axis starting at .260 makes a .312 vs .290 difference look enormous. Starting at zero shows the true proportion.
  • 02 Label your axes - always. A chart without axis labels forces the reader to guess. Every axis needs a label and units. "Salary" is incomplete; "Annual Salary (USD)" is correct.
  • 03 Use color purposefully, not decoratively. Color should encode information (category, magnitude, emphasis) - not just make the chart look nice. Use a consistent, colorblind-friendly palette. Seaborn's default palettes are designed for this.
  • 04 Reduce clutter. Remove every element that does not add information: unnecessary gridlines, decorative borders (spines), redundant legends. Edward Tufte called this maximizing the "data-ink ratio."
  • 05 Choose the right chart for the data type. Use a bar chart for categorical comparisons, a line chart for time series, a scatter for two continuous variables, a histogram to show distribution. The chart type determines what pattern the reader's eye will find.
  • 06 Show uncertainty. If you are plotting a mean, show the confidence interval or standard error. A point estimate without uncertainty hides important information about the reliability of the finding.
  • 07 Write a descriptive title. "Sales Data" is not a title; "Q3 Sales Declined 12% vs. Prior Year" is. A good title tells the reader the takeaway, not just the topic.
  • 08 Use plt.tight_layout(). This prevents labels and subplots from overlapping. Make it a habit to call it before every plt.show() or plt.savefig().
Saving publication-quality figures
plt.savefig('chart.png', dpi=150, bbox_inches='tight') - the dpi parameter controls resolution (150 is good for presentations, 300 for print). bbox_inches='tight' prevents labels from being clipped. For vector graphics (scalable at any size), save as .svg or .pdf instead.
